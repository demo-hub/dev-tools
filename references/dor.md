# Definition of Ready

---

## 🟦 User Stories / PBIs

### Business & Customer Understanding

- [ ] Clear goal is documented
- [ ] Business context is documented
- [ ] Customer/persona identified
- [ ] Roles, profiles, and permissions known
- [ ] The reason/need ("why") is understood

### Requirements & Acceptance Criteria

- [ ] Acceptance criteria exist
- [ ] Acceptance criteria are clear and understandable
- [ ] Happy path is explicitly covered in AC
- [ ] Edge scenarios are listed
- [ ] Negative scenarios are listed
- [ ] Error-handling behavior is defined
- [ ] Preconditions are stated where relevant

### Data & Dependencies

- [ ] Required data contracts are known
- [ ] Required data contracts are received (internal or external teams)
- [ ] Test data described OR steps to create it documented
- [ ] Team-to-team dependencies identified
- [ ] System-to-system dependencies identified
- [ ] All dependencies clarified
- [ ] All dependencies delivered

### Design & Architecture

- [ ] UI/UX designs finalized
- [ ] UI/UX mockups confirmed
- [ ] UI/UX accepted by responsible teams
- [ ] Architecture design completed (if required)
- [ ] Technical approach approved
- [ ] Tech Lead understands and approves the technical approach
- [ ] Engineering team understands the technical approach
- [ ] Technical detail session completed by engineers/tech leads

### Infrastructure & Access

- [ ] All required infrastructure is implemented
- [ ] All necessary access granted to team members

### Planning & Sizing

- [ ] All development tasks identified and listed
- [ ] Story is estimated
- [ ] Story fits within a single sprint

---

## 🟩 Bugs

- [ ] Evidence provided (e.g. case number, screenshots, logs)
- [ ] Step-by-step reproduction process included
- [ ] Current behavior clearly described
- [ ] Expected behavior clearly described
- [ ] Operating system specified
- [ ] Browser and version specified
- [ ] Related bugs referenced
- [ ] Timestamps provided for log/trace investigation
- [ ] Application version included (if detected in PP)
- [ ] Bug is estimated
